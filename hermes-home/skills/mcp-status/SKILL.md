# MCP Status — статус MCP-серверов Hermes

## Когда использовать

Пользователь хочет увидеть, какие MCP-серверы установлены, у каких есть ключи и какие
реально активны. Триггеры: «покажи MCP сервера», «статус MCP», «какие MCP активны»,
«list/show mcp servers».

## Что делает

Скрипт `scripts/mcp_status.py` читает:
- `<HERMES_HOME>/config.yaml` → секция `mcp_servers` (что **установлено**);
- `<HERMES_HOME>/.env` → **наличие** ключей (печатаются только имена переменных, никогда значения);
- `<HERMES_HOME>/logs/agent.log` → какие серверы **реально зарегистрировались** (активны) и сколько инструментов подняли.

## Как выполнять

1. Запусти скрипт (HERMES_HOME определяется автоматически, на Windows — из `%LOCALAPPDATA%\hermes`):

   ```bash
   python "<HERMES_HOME>/skills/mcp-status/scripts/mcp_status.py"
   ```

   На Windows, если в консоли мусор вместо кириллицы, добавь `PYTHONUTF8=1`.

2. Покажи пользователю вывод скрипта **как есть** (это готовая таблица): сервер, тип,
   статус ключа, активность. Ничего не дописывай и не выдумывай цифры.

## Важно про безопасность

- Скрипт НЕ выводит значения ключей — только факт наличия. Не раскрывай секреты сам.
- Если у сервера статус «✗ нет ключа» — подскажи, какую переменную добавить в `.env`.
- Если сервер «❌ не подключился» при наличии ключа — предложи `/reload-mcp` или перезапуск gateway.
