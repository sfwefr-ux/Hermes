## Добавление записей

### Финансы
Формат от пользователя: «Расход/Заработал СУММА, КАТЕГОРИЯ/ОПИСАНИЕ»
→ Создать запись в базе «Финансы BadLock» через `notion_create_pages` с parent data_source_id.

### Модель по умолчанию
Все рутинные задачи (финансы, монтажи, ответы клиентам) — **DeepSeek v4 pro**. Claude Sonnet включается только при автофолбэке. После фолбэка сессия «прилипает» к Sonnet — это не ошибка. Если Станислав спрашивает — объяснить: переключение обратно происходит автоматически с нового запроса/сессии, ручного действия не нужно.

**Каскад фолбэков при записи (в порядке приоритета):**

1. **Notion MCP** — основной путь. `notion_create_pages` с parent `data_source_id`.
2. **Локальный backup** — если Notion недоступен (инструменты отсутствуют, `keepalive failed`, `TaskGroup`):
   → Попробовать `write_file` в `<HERMES_HOME>/badlock-expenses-backup.md`.
   → Если `write_file` сломан (WSL/bash), попробовать `execute_code` с `write_file()` из `hermes_tools`.
   → Если `execute_code` заблокирован (approvals.cron_mode) — создать файл через `patch` (только если файл уже существует; иначе patch не сработает на новом файле).
3. **Memory** — если ВСЕ методы записи недоступны:
   → Сохранить запись в `memory` (target=`memory`) с пометкой «Ожидают записи» + дата.
   → Сообщить пользователю: что именно не записалось, причина, сумма.
   → При восстановлении инструментов — перенести из memory в Notion/backup и удалить memory-запись.

Детали: `notion-business-ops` → pitfall «Notion MCP упал mid-session».

**Pitfall — дубликаты при массовом создании:** если пользователь в нескольких голосовых сообщениях упоминает одного клиента/заказ — НЕ создавать отдельную запись на каждое упоминание. Проверять: одно и то же имя + сумма + дата → один заказ, одна запись. При сомнении — уточнить у пользователя.

**Pitfall — дата после перезапуска:** после gateway restart или `/restart` сессия может потерять контекст текущей даты. Всегда проверять реальную дату (разговор начат: смотреть в conversation metadata) перед созданием записей. Не полагаться на контекст прерванной сессии.

**Формат сводки финансов:** при запросе «сводка за последние дни»:
- Группировать по дням: заголовок `### ДД МЕСЯЦА (день_недели)`
- Каждая запись строкой: `Описание: **±СУММА ₽**`
- Итого за день: `*Итого:* **±СУММА ₽**`
- В конце: **Доход: X ₽ | Расход: Y ₽ | Прибыль: Z ₽**
- Кратко, без таблиц, без лишних слов.

### Монтажи
Формат: «Запиши монтаж НАЗВАНИЕ_РАБОТ на ДАТА от КЛИЕНТ»
→ Создать запись в «График монтажей». Статус по умолчанию «Подтверждён».

### ⚠️ Pitfall: даты в Notion требуют развёрнутого формата

Свойства типа Date НЕЛЬЗЯ передавать как `"Дата": "2026-06-19"`. Notion API требует развёрнутый формат:
```
"date:Дата:start": "2026-06-19",
"date:Дата:is_datetime": 0
```
Без этого — ошибка `validation_error: Date type must be expanded`. Это касается ВСЕХ date-полей в финансах и графике. При первом вызове всегда использовать развёрнутый формат, не пытаться передать просто строку даты.
