# Notion Business Operations

Шаблоны и workflows для создания операционных баз в Notion через MCP Market: учёт финансов, базы знаний, скрипты общения с клиентами.

## Предварительные условия

- Notion MCP подключён через MCP Market (см. `productivity/notion`)
- OAuth-токен в конфигурации Hermes
- Инструменты: `mcp_mcpmarket_notion_notion_*`

## Финансовый учёт

### Создание базы

```sql
CREATE TABLE (
  "Название" TITLE,
  "Тип" SELECT('Расход':red, 'Доход':green),
  "Сумма" NUMBER,
  "Категория" SELECT(<варианты>),
  "Дата" DATE,
  "Комментарий" RICH_TEXT
)
```

Категории расходов: Материалы, Инструмент, Транспорт, Реклама, Связь, Аренда, Личное, Прочее.
Категории доходов: Установка замка, Вскрытие, СКУД, Видеонаблюдение, Домофон, Прочее.

### Добавление записи

Через `notion_create_pages` с `parent: {data_source_id: "..."}`. Поля даты требуют развёрнутого формата:
- `date:Дата:start` — ISO-8601
- `date:Дата:is_datetime` — 0 для даты, 1 для datetime

### Расширение категорий

`notion_update_data_source` → `ALTER COLUMN "Категория" SELECT(...)` с полным списком опций.

### Ежемесячная сводка (cron)

```yaml
schedule: "0 9 1 * *"
skills: ["productivity/notion"]
enabled_toolsets: ["web", "terminal", "file", "skills"]
```

Промпт: запросить все записи из data_source за прошлый месяц, посчитать итоги (доход, расход, прибыль, топ-категории), отправить пользователю.

## База знаний и скрипты продаж

### Массовая загрузка контента

```python
notion_update_page(
    page_id="...",
    command="insert_content",  # НЕ ЗАБЫВАТЬ command!
    content="## Заголовок\n\nmarkdown...",
    position={"type": "end"}
)
```

**Pitfall:** параметр `command` обязателен при вызове `notion_update_page`. Без него — ошибка валидации.

### Структура страницы знаний

Рекомендуемая структура для базы знаний малого бизнеса:
1. Общая информация (бизнес, контакты, график)
2. Прайс-лист
3. FAQ
4. Скрипт общения с клиентом
5. Работа с возражениями — 12 типов по 50 ответов:
   - Нет времени на это сейчас
   - У нас уже это есть
   - Я подумаю
   - Нам это не нужно
   - Нет бюджета
   - Мы уже работаем с вашим конкурентом
   - Мне нужно обсудить это с командой
   - У нас уже есть похожее решение
   - Я не верю, что это сработает
   - Слишком долгое время доставки
   - Могу найти дешевле у другого поставщика
   - Не уверен в качестве
6. Ссылки
7. Стратегия продвижения (дочерняя страница)

## Коммуникация с пользователем

- Пользователь общается на русском языке
- Предпочитает краткость и действия вместо долгих объяснений
- Часто отправляет голосовые сообщения — обрабатывать как русскую речь
- Называет ассистента «Генрих»
- При загрузке контента: минимум подтверждений, максимум скорости

## Pitfalls

### Не делать бизнес-допущений без подтверждения

При описании услуг/структуры бизнеса НЕ добавлять детали, которые пользователь не подтвердил:
- Не называть что-то «стандартом» без явного указания пользователя
- Не выделять услугу как отдельную, если пользователь рассматривает её как часть другой (пример: доводчик — часть установки магнитного замка, не отдельная услуга)
- Переспрашивать перед внесением структурных изменений
- Если пользователь поправляет формулировку 2+ раза — перестать интерпретировать и действовать строго по его словам

### MCP-сервер падает после 3+ ошибок 400

Если `notion_update_page` с `update_content` трижды подряд возвращает "No matches found", MCP-сервер может стать недоступным. **Ждать 60 секунд**, затем повторить с другим (более коротким) фрагментом old_str. Лучше заранее избегать этой ситуации — всегда делать `notion_fetch` перед `update_content` и использовать `insert_content` для добавления новых разделов (он не требует точного совпадения и не падает).

**Восстановление после отключения MCP-сервера:**
1. Подождать 60 секунд — сервер может восстановиться сам
2. Если всё ещё недоступен — нужен `/restart` Hermes или перезапуск
3. После рестарта проверить наличие инструментов через `notion_fetch`

### ALTER COLUMN на SELECT — перечислить ВСЕ опции

При изменении SELECT-свойства через `notion_update_data_source` с `ALTER COLUMN "Имя" SET SELECT(...)`, нужно перечислить **все существующие опции** + новые. Инструмент заменяет весь список, а не добавляет. Пропущенная опция удаляется молча.

### Проверка дубликатов при загрузке возражений

При массовой загрузке ответов на возражения — проверять заголовок перед добавлением:
- Сверить с уже загруженными (прокрутить список в голове или fetch страницу)
- Если возражение уже есть — сообщить пользователю, не добавлять повторно
- Пользователь может не помнить, что уже загружал — это нормально

### Notion update_content: точное совпадение old_str

При вызове `notion_update_page` с `command="update_content"` — old_str должен точно совпадать с текстом в документе. Перед вызовом всегда делать `notion_fetch` для получения актуального контента и копировать текст дословно.

**Предпочтение:** для добавления новых разделов всегда использовать `insert_content` (position end) вместо `update_content` — не требует точного совпадения и не вызывает каскадных ошибок 400. `update_content` оставить только для замены существующего текста.

### Notion MCP-инструменты не загружены в сессию

Если `notion_search`, `notion_fetch` и другие инструменты Notion MCP отсутствуют в текущей сессии, хотя сервер `mcpmarket_notion` прописан в `config.yaml`:

1. **Проверить токен:** в `mcp_servers.mcpmarket_notion.headers.Authorization` должен быть **JWT Bearer-токен**, а не `sk_user_...` (ключ аккаунта MCP Market для веб-интерфейса не подходит для MCP-подключений). Подробнее: `devops/hermes-mcp-setup` → раздел «Auth methods that DO NOT work for MCP Market».
2. **Проверить `Accept`-заголовок:** Notion MCP требует `Accept: application/json, text/event-stream`. Без него — 406.
3. Если токен верный, но инструментов нет — сервер не стартанул при запуске Hermes. Нужен `/restart` или перезапуск.

**Резервный способ прочитать конфиг** (если терминал и `read_file` недоступны):
- `browser_navigate(url="file:///C:/Users/1/AppData/Local/hermes/config.yaml")`
- `browser_console(expression="document.body.innerText")` — вернёт полный YAML
- Работает в том числе из cron-сессии.

### Параметр command обязателен

`notion_update_page` требует параметр `command`. Без него — ошибка валидации. Всегда указывать: insert_content, update_content, update_properties, replace_content и т.д.
